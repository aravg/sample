🖥️ Streamlit Web Application
User fills trip planning form: source city, destination, days, travelers, total budget (₹), travel type, food preferences, interests. Clicks "Generate My Trip Plan".
Streamlit · Dark Glassmorphism UI
8 UI Tabs after results: Overview · Hotels · Transport · Itinerary · Budget · Places · PDF · Observability.
Custom CSS targets data-baseweb selectors for dark input widgets.
🛡️ Pre-flight Content Guard
InputGuardrails.validate_form_data() scans all text fields against 12 policy categories using compiled regex named-group patterns. Instant — no LLM call.
Regex · Instant · No LLM call
Categories: Violence/Threat · Weapons · Illegal Activity · Controlled Substances · Fraud/Scam · Terrorism · Hate Speech · Profanity · Abusive Language · Explicit Content · Self-Harm · Spam

Scans: query, source, destination, user_name, food_preferences, interests, travel_type
🚫 Content Violation Detected
Workflow is blocked. Red dialog displayed. All result sections blurred with lock overlay. No PDF, no plan generated.
passed = False · Workflow Halted
🔍 Observability (Blocked View)
Shows each flagged term with its violation category tag and remediation tip. Only visible section — all others blurred.
Flagged words · Category · Fix hint
🧠 Orchestrator Agent
LLM-powered routing hub. Reads TripState, decides next agent. Validates decisions, prevents loops, enforces max 3 retries per agent. Falls back to rule-based logic if LLM fails.
Claude API · JSON routing · State-aware
Routes: user_input → memory_retrieval → gather_data → budget → itinerary → review → memory_update → pdf_generator → end

Decision schema: {next_step, reason, conflicts[], retry_agent}
📋 User Input Agent
LLM extracts structured trip preferences from natural language query. Applies InputGuardrails.sanitize() then validate(). Stores result in guardrail_results["input"].
Claude · JSON extraction · Fallback parser
Outputs trip_preferences: {source, destination, days, travelers, budget, travel_type, hotel_preference, food_preferences, transport_preference, interests, dates{start,end,display}, luxury_level}
🗄️ Memory Retrieval Agent
Queries FAISS vector store for past trips and user preferences. Personalises plan based on travel history.
FAISS · Vector similarity · data/vector_store/
Reads data/vector_store/{user_id}/documents.json. Populates: past_trips[] and user_preferences_memory{} in TripState.
⚡ gather_data — sequential: Weather → Transport → Hotels → Places
🌤️ Weather Agent
Forecast, seasonal advice, packing tips for destination dates
OpenWeather API
Outputs weather_data: {forecast, temperature, conditions, seasonal_tips, packing_list, weather_alerts, summary}
✈️ Transport Agent
Flights & trains inter-city. Metro / bus / rail for same-city trips.
Smart city detection
is_same_city() normalises names & aliases. Same city → search_local_transport() for 7 cities. Different city → flights + trains.
🏨 Hotel Agent
Finds & ranks hotels by budget tier, rating, proximity to attractions
Hotel search tools
Outputs hotel_data: {recommendations[], best_pick{name, price_per_night, rating, location}, price_range, filters_applied}
📍 Places Agent
Attractions, restaurants, activities filtered by user interests
Places / search API
Outputs places_data: {attractions[], restaurants[], activities[], hidden_gems[], categorised by interest tags}
💰 Budget Agent
Calculates INR breakdown: hotels, transport, food, activities, misc. Hard cap: if total exceeds budget, all categories are proportionally scaled down.
INR · Hard cap · Proportional scaling
Hard cap: estimated_total > budget → scale = budget/total → applied to all categories.

Outputs budget_summary: {total_inr, breakdown{hotels,transport,food,activities,misc}, forced_within_budget, per_person_cost}
📅 Itinerary Agent
Generates day-by-day plan: activities, timings, restaurant picks, travel notes, daily budgets. Uses weather, places, and transport data.
Claude · Day-wise planning
Outputs itinerary: {days[{day_number, date, theme, morning, afternoon, evening, dinner, notes, activities[], daily_budget_inr}], highlights[], total_days}
🔎 Review Agent
Validates: budget compliance, itinerary completeness (all days covered), hotel fit, weather suitability, transport feasibility.
Claude · Conflict detection · Rule fallback
Returns review_status: {approved, issues[], conflicts[], retry_agent (hotel|budget|itinerary|transport), review_notes, confidence_score 0-100}

Auto-approves after 3 retries.
🔄 Retry Failed Agent
Re-runs: budget · itinerary · hotel · transport. Orchestrator routes back to review after retry.
max 3× · retry_counts tracked
retry_counts stored per agent in TripState. After retry, routes back to review_node for re-evaluation. After 3 retries the review auto-approves with a note.
⚖️ Approval Decision Gate
Approved → Memory Update. Issues found → retry the failing agent (max 3× per agent). Max retries reached → auto-approve.
Conditional routing · retry_counts ≤ 3
💾 Memory Update Agent
Persists this trip's preferences and outcomes into the FAISS vector store for future personalisation of the same user.
FAISS · Persistent · Per-user
📄 PDF Generator Agent
Generates A4 travel report in INR. Runs Output Guardrails + RAGAS + DeepEval before generating. Saves to output/TripPlan_{dest}_{timestamp}.pdf.
ReportLab · All costs in ₹ · A4
PDF sections: header banner, trip summary, weather, hotels, transport, day-wise itinerary, full budget breakdown — all in ₹.

Path stored in TripState.pdf_path.
🛡️ Output Guardrails
5 checks: budget compliance (≤105%), itinerary completeness, hotel availability, transport availability, content quality (empty days).
OutputGuardrails.validate()
Stores result in guardrail_results["output"]. Budget check: estimated_cost ≤ total_budget × 1.05. Completeness: itinerary_days ≥ requested_days.
